# Password Manager

Клиент-серверное приложение для безопасного хранения паролей и конфиденциальных данных.

## Архитектура

```
┌─────────────────┐         gRPC          ┌─────────────────┐
│   Client CLI    │ ◄─────────────────►   │   Server        │
│  (Go/Cobra)     │    Синхронизация      │  (Go/gRPC)      │
└─────────────────┘                         └─────────────────┘
       │                                            │
       ▼                                            ▼
┌─────────────────┐                         ┌─────────────────┐
│  BoltDB         │                         │  PostgreSQL     │
│                 │                         │                 │
└─────────────────┘                         └─────────────────┘
```

## Возможности

- ✅ Шифрование данных на клиенте (AES-256-GCM)
- ✅ Синхронизация между устройствами
- ✅ Поддержка нескольких типов записей (логины, текст, бинарные данные, карточки)
- ✅ Теги для фильтрации записей
- ✅ JWT-аутентификация
- ✅ Версионирование для разрешения конфликтов синхронизации

## Установка

```bash
# Сборка клиента
make build-client

# Сборка сервера
make build-server
```

Клиент может быть сгенерирован под разные ос, более подробную информацию (доступные возможности) можно получить сделав
```bash

make help
```

## Использование

```bash
# Вывод подсказки по возможностям клиента
client --help
```

При первом подключении клиент получает сертификат сервера и дальше хранит его хэш для проверки валидности.
С помощью переменной окружения PM_TEST_MODE можно включить тестовый режим для клиента (соединение становится insecure)

## Технологии

- **Go 1.24+**
- **gRPC** — межпроцессное взаимодействие
- **BoltDB** — локальное хранилище
- **PostgreSQL** — серверная БД
- **Cobra** — CLI фреймворк
- **Protobuf** — схема данных
